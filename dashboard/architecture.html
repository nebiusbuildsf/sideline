Perception to Action

How a sensor observation becomes a robot action

When the MentorPi's camera sees a ball land out, here's the full chain from sensor to actuator.
👁

Sensor Captures

Camera sensor captures frame. Depth camera adds spatial context. Lidar maps court geometry.

→
🧠

Agent Reasons

VLM: "Ball landed 3cm past baseline. Rule: ball outside line is out. Call: OUT. Confidence: 0.92"

→
⚡

Tools Dispatched

announce_call("out", "Out!") + update_score("p2") + robot_gesture("point_out", "right")

→
🔌

ROSA Translates

Natural language → ROS2: /speaker "Out!", /servo [60°, 30°, 45°], /cmd_vel toward play

→
🤖

Actuators Execute

Speaker announces "Out!" Arm extends right. Wheels rotate toward the baseline.

Protocol Details

Three protocols in depth

Each protocol serves a different purpose. Together they make the agent truly agentic.
⚡
Function Calls
Native tool use via Nebius API
  • 1
    Define tools — update_score, announce_call, robot_gesture, no_call
  • 2
    Send frame + tools to Nebius VLM with tool_choice="auto"
  • 3
    Model decides which tools to call based on what it sees
  • 4
    Execute — parse tool_calls, run each function
// Model response tool_calls: [ {"name": "announce_call", "args": {"call_type": "out", "announcement": "Out!", "confidence": 0.92}}, {"name": "update_score", "args": {"player": "p2", "reason": "out"}}, {"name": "robot_gesture", "args": {"gesture": "point_out", "direction": "right"}} ]
🔌
MCP
Model Context Protocol — external tools
  • 1
    MCP Server exposes tools: analyze_frame, get_score, lookup_rule, robot_action
  • 2
    Any MCP client (Claude, ChatGPT, Cursor) connects and discovers tools
  • 3
    Client calls tool — e.g. "Was that ball out?" → analyze_frame()
  • 4
    Swap backends — change robot, change rules DB, agent code stays the same
# MCP Server topology sideline-referee stdio://localhost ├── analyze_frame(image, sport) ├── get_score() ├── get_call_history(last_n) ├── lookup_rule(sport, situation) └── robot_action(action, params) tavily-search api.tavily.com └── search(query) robot-control ros2://mentorpi ├── speak(text) ├── gesture(type, direction) └── move(direction, speed)
🔗
A2A — Agent-to-Agent Protocol
Multi-agent coordination via JSON-RPC 2.0
  • 1
    AgentCard published at /.well-known/agent.json — skills, capabilities, endpoint
  • 2
    Discovery — agents find each other, read capabilities
  • 3
    tasks/send — create a task, agent processes it (submitted → working → completed)
  • 4
    Streaming — SSE for real-time updates as tasks progress

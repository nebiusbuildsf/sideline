Protocol Layer

Three ways to dispatch actions

Each protocol adds a layer of capability. Function calls are fast and internal. MCP makes tools swappable. A2A enables multi-agent coordination.

bolt Function Calls

OpenAI tool_use · Nebius API

VLM decides which tools to invoke. Score updates, announcements, gestures — all through native tool calling.

tool_calls: [ {name: "announce_call", args: {call: "out", text: "Out!", confidence: 0.92}}, {name: "update_score", args: {player: "p2"}}, {name: "robot_gesture", args: {gesture: "point_out"}} ]

extension MCP Server

Model Context Protocol

Expose referee as MCP tools. Any client — Claude, ChatGPT, Cursor — can ask "was that ball out?" and get a referee analysis.

sideline-referee (stdio) ├── analyze_frame(img, sport) ├── get_score() ├── lookup_rule(sport, situation) └── robot_action(cmd, params) tavily-search (https) └── search(query) robot-control (ros2) ├── speak(text) ├── gesture(type) └── move(dir, speed)

hub A2A Protocol

Agent-to-Agent · JSON-RPC 2.0

Multiple Sideline agents coordinate. Three cameras, three agents, one coordinator — autonomous VAR.

Camera-1 Agent ──┐ ├─▶ Coordinator Camera-2 Agent ──┘ │ ├─▶ Commentator (majority vote) │ └─▶ Robot Agent → gesture → speak
